# Shuttle Finder — Spec

## Overview

출발 정류장과 도착 정류장을 선택하면, 해당 방향의 다음 셔틀 3개와 각각의 남은 시간을 즉시 보여주는 웹앱.  
로그인 없음. 정적 시간표 기반. 평일 전용.

---

## Users

- 회사 직원 (인증 없이 URL 북마크로 접근)

---

## Scenarios

### 1. 정상 조회 — 다음 셔틀 존재

**Given** 평일이고 현재 시각 이후 운행 예정인 셔틀이 1개 이상 존재한다  
**When** 출발 정류장과 도착 정류장을 선택한다  
**Then**
- 해당 방향의 다음 셔틀을 최대 3개 표시한다
- 각 셔틀마다 출발 시각과 남은 시간(분)을 함께 표시한다
- 남은 시간이 1분 미만인 셔틀은 "곧 출발"로 표시한다

---

### 2. 오늘 마지막 셔틀 이후 조회

**Given** 평일이고 오늘 해당 방향의 모든 셔틀이 이미 출발했다  
**When** 출발 정류장과 도착 정류장을 선택한다  
**Then**
- "오늘 운행 종료"를 표시한다
- 다음 운행일(다음 평일)의 첫 셔틀 시각을 안내한다

---

### 3. 주말 또는 공휴일 접속

**Given** 오늘이 주말 또는 공휴일이다  
**When** 앱에 접속한다  
**Then**
- "오늘은 운행하지 않습니다"를 표시한다
- 다음 평일의 첫 셔틀 시각을 안내한다

---

### 4. 정류장 미선택 상태

**Given** 출발 또는 도착 정류장 중 하나라도 선택되지 않았다  
**When** 결과 영역이 표시된다  
**Then**
- 셔틀 시간 정보를 표시하지 않는다
- 정류장 선택을 유도하는 안내 문구를 표시한다

---

### 5. 출발지와 도착지가 동일한 경우

**Given** 사용자가 출발 정류장과 도착 정류장을 동일하게 선택했다  
**When** 두 번째 정류장을 선택하는 순간  
**Then**
- 해당 선택이 불가하도록 막는다 (동일 정류장은 목록에서 비활성화 또는 제외)

---

## Invariants

- 현재 시각은 사용자 기기의 로컬 시간을 기준으로 한다
- A→B 노선과 B→A 노선의 시간표는 독립적으로 관리된다 (양방향 시간 상이)
- 공휴일 판단은 앱이 명시적으로 관리하는 날짜 목록 기준이다

---

## Out of Scope (MVP)

| 항목 | 제외 이유 |
|---|---|
| 로그인 / 인증 | 내부 직원 전용, 민감 데이터 없음 |
| GPS 자동 위치 감지 | 실내·지하 오류율 높음. 정류장 선택으로 충분 |
| 실시간 셔틀 위치 | 차량 단말 인프라 필요, 핵심 가치 검증 후 검토 |
| 좌석 예약 | 현재 수요 미확인 |
| 푸시 알림 | 웹앱 채택 먼저 검증 |
| 환승 경로 최적화 | 단일 구간 이동이 주 시나리오 |
| 어드민 시간표 관리 | 시간표가 거의 안 바뀌므로 코드 수정으로 관리 |
