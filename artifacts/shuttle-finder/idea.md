# Shuttle Finder — Idea

## HMW 문제 진술

> "직원이 근무사이트 간 이동할 때, 엑셀/PDF 시간표를 뒤지는 대신 출발지·목적지만 고르면 '지금 뛰어야 하나, 여유 있나'를 즉시 알 수 있을까?"

시간표 탐색 문제가 아니라 **판단 속도** 문제다.

---

## 컨텍스트

| 항목 | 내용 |
|---|---|
| 시간표 관리 | 엑셀/PDF 고정 배포 (거의 안 바뀜) |
| 주 사용 시나리오 | 근무사이트 간 업무 이동 |
| 위치 입력 | 정류장 목록에서 선택 |
| 핵심 고통 | 시간표 찾다가 이미 셔틀이 떠났을 때 |

---

## MVP One-liner

> **출발 정류장 + 도착 정류장을 고르면, 다음 셔틀까지 남은 시간을 즉시 보여주는 웹앱**
> 정적 시간표 기반. 설치 불필요. 북마크 하나로 완결.

---

## 숨은 가정

| 분류 | 가정 | 검증 방법 |
|---|---|---|
| **Must Be True** | 앱 시간표와 실제 운행 시간이 일치한다 | 배포 전 운수팀과 시간표 교차 확인 |
| **Should Be True** | 직원이 앱을 북마크해 습관적으로 쓴다 | 홈 화면 URL 공유, 첫 2주 사용률 측정 |
| **Might Be True** | 모바일 UX 없으면 불편하다 | 웹앱으로 먼저 검증, 반응 보고 결정 |
